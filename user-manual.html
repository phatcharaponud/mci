1. การ Login เข้าใช้งานระบบ

ระบบใช้การยืนยันตัวตนผ่าน Google Account ที่เป็นอีเมล @up.ac.th เท่านั้น พร้อมระบบ PIN สำหรับความปลอดภัยเพิ่มเติม

1.1 การเข้าสู่ระบบครั้งแรก (ลงทะเบียน)

1เปิดเว็บไซต์ระบบที่ cm-uph.web.app

2กดปุ่ม "เข้าสู่ระบบด้วยอีเมล @up.ac.th"

3เลือก Google Account ที่เป็น @up.ac.th

4ระบบจะแสดงหน้าลงทะเบียน กรอกข้อมูล:

  • ชื่อ — ชื่อจริง
  • นามสกุล — นามสกุล
  • หน่วยงาน — พิมพ์ค้นหาหน่วยงานจากรายการ

5กดปุ่ม "ลงทะเบียน"

6ระบบจะแสดงหน้า "รอการอนุมัติ" — รอให้ Admin อนุมัติบัญชีก่อน

หมายเหตุ: ผู้ใช้คนแรกที่ตั้งเป็น ADMIN_EMAIL จะได้รับสิทธิ์ Admin โดยอัตโนมัติ ไม่ต้องรออนุมัติ

1.2 การตั้ง PIN (ครั้งแรกหลังได้รับอนุมัติ)

1หลังจาก Admin อนุมัติแล้ว เมื่อ Login จะแสดงหน้า "ตั้ง PIN"

2กรอก PIN ตัวเลข 4-6 หลัก

3กรอก PIN อีกครั้งเพื่อยืนยัน

4กดปุ่ม "ตั้ง PIN"

ข้อห้าม: ไม่อนุญาต PIN ที่ง่ายเกินไป เช่น 1111, 1234, 0000

1.3 การเข้าสู่ระบบครั้งถัดไป

1กดปุ่ม "เข้าสู่ระบบด้วยอีเมล @up.ac.th"

2เลือก Google Account

3กรอก PIN ที่ตั้งไว้

4กดปุ่ม "เข้าสู่ระบบ"

ความปลอดภัย: ใส่ PIN ผิด 5 ครั้งติดต่อกัน → บัญชีจะถูกล็อค 15 นาที หากลืม PIN ให้ติดต่อ Admin เพื่อ Reset PIN

2. การบันทึกข้อร้องเรียนใหม่

การบันทึกข้อร้องเรียนเป็นหน้าที่ของ CRM เจ้าหน้าที่ศูนย์รับข้อร้องเรียน และ Admin

2.1 ขั้นตอนการบันทึก

1กดเมนู "บันทึกข้อร้องเรียน" ที่แถบด้านซ้าย หรือกดปุ่ม "+ บันทึกใหม่"

2เลือก วันที่ รับเรื่อง

2.2 กรอกข้อมูลในแบบฟอร์ม

ส่วนที่รายละเอียด
1. รายละเอียดผู้ร้องเรียน - ชื่อ, สกุล, ที่อยู่, โทรศัพท์ (หรือติ๊ก "ไม่ประสงค์แจ้งชื่อ")
- สถานะ: ค่าเริ่มต้น "ผู้ป่วย/ผู้รับบริการ" กดแก้ไขเพื่อเปลี่ยน
- วิธีการร้องเรียน: เลือกได้หลายช่องทาง (จดหมาย, โทรศัพท์, Line OA ฯลฯ)
2. หน่วยงานที่ถูกร้องเรียนพิมพ์ค้นหาและเลือกจากรายการหน่วยงาน
3. เรื่องที่ร้องเรียนกรอกหัวข้อเรื่อง และรายละเอียด
ประเมินระดับความรุนแรง - ระดับ 1: บ่น/เสนอแนะ (กรอบเวลา 3 วันทำการ)
- ระดับ 2: ตำหนิ/ร้องทุกข์ (กรอบเวลา 72 ชั่วโมง)
- ระดับ 3: ด่าทอ/ขู่ฟ้อง/ทำร้าย (กรอบเวลา 6 ชั่วโมง)
การดำเนินการศูนย์ - รายละเอียดการดำเนินการ
- ชื่อเจ้าหน้าที่ผู้บันทึก
- Tag หน่วยงาน / Tag ผู้รับผิดชอบ (แจ้งให้ทราบ)
*สามารถพิมพ์ชื่อผู้รับผิดชอบเองได้ แม้ไม่มีในรายการ

3กดปุ่ม "บันทึกข้อร้องเรียน"

แจ้งเตือน LINE: เมื่อบันทึกสำเร็จ ระบบจะส่งแจ้งเตือนไปยังกลุ่ม LINE OA โดยอัตโนมัติ

3. การบันทึกการตรวจสอบ/ตอบกลับ และผลการแก้ไข

เป็นหน้าที่ของ User หน่วยงานที่ถูก Tag ให้ดำเนินการตรวจสอบและบันทึกผลการแก้ไข

3.1 การบันทึกการตรวจสอบ/ตอบกลับ

1เข้าเมนู "ทะเบียนข้อร้องเรียน"

2หาเรื่องที่ต้องการ → กดปุ่ม "ตรวจสอบ"

3กรอกข้อมูลในแท็บ "ตรวจสอบ/ตอบกลับ":

  • 1. ตรวจสอบข้อเท็จจริง
  • 2. วิเคราะห์สาเหตุ (Root Cause Analysis)
  • 3. แนวทางแก้ไขและป้องกัน
  • ผู้ตอบ
  • วันที่ตรวจสอบ/ตอบกลับ (ปฏิทิน พ.ศ.)

4กดปุ่ม "บันทึกผลการตรวจสอบ"

3.2 การบันทึกผลการแก้ไข

1จากรายการเดียวกัน กดแท็บ "ผลการแก้ไข"

2กรอกข้อมูล:

  • การดำเนินการแก้ไข
  • ผลการแก้ไข
  • มาตรการป้องกันการเกิดซ้ำ
  • วันที่แก้ไขแล้วเสร็จ (ปฏิทิน พ.ศ.)

3กดปุ่ม "บันทึกผลการแก้ไข"

สถานะอัตโนมัติ: เมื่อบันทึกการตรวจสอบ สถานะจะเปลี่ยนเป็น "กำลังดำเนินการ" และเมื่อบันทึกผลแก้ไข สถานะจะเปลี่ยนเป็น "แก้ไขแล้ว" โดยอัตโนมัติ

4. การใช้งานทะเบียนข้อร้องเรียน

ทะเบียนข้อร้องเรียนเป็นหน้ารวมรายการทั้งหมด สามารถค้นหา กรอง และ Export ได้

4.1 มุมมอง

สามารถสลับมุมมองได้ 2 แบบ:

  • มุมมองตาราง — แสดงรายการเป็นตารางแถว เหมาะกับหน้าจอใหญ่
  • มุมมองการ์ด — แสดงเป็นการ์ด เหมาะกับมือถือ

4.2 การค้นหาและกรอง

ตัวกรองรายละเอียด
ช่องค้นหาพิมพ์คำค้นหาจากหัวข้อ, รายละเอียด หรือรหัส
สถานะกรองตาม: กำลังตรวจสอบ, กำลังดำเนินการ, แก้ไขแล้ว
ระดับความรุนแรงกรองตาม: ระดับ 1, 2 หรือ 3
หน่วยงานพิมพ์เลือกหน่วยงาน
วันที่เลือกเดือน หรือกำหนดช่วงวัน (จาก-ถึง)

4.3 การ Export PDF

1ในคอลัมน์ "สรุปรายงาน" กดปุ่ม "Export PDF" ของรายการที่ต้องการ

2ระบบจะเปิดหน้าต่างใหม่แสดงรายงาน

3หน้าต่าง Print จะเปิดอัตโนมัติ → เลือก "Save as PDF" หรือ "Print"

เนื้อหา PDF: รายงานประกอบด้วยข้อมูลทั่วไป, เรื่องที่ร้องเรียน, การดำเนินการศูนย์, ผลการสอบสวน (ถ้ามี) และผลการแก้ไข (ถ้ามี)

5. การใช้งานปฏิทิน

ปฏิทินแสดงภาพรวมข้อร้องเรียนและการนัดประชุมในแต่ละวัน ช่วยติดตามการดำเนินการ

5.1 การดูข้อมูลในปฏิทิน

  • จุดสีเหลือง/เขียว/แดง = ข้อร้องเรียน (สีตามระดับความรุนแรง)
  • จุดสีม่วง = นัดประชุม
  • ในแต่ละวันจะแสดงข้อความสรุปเรื่องข้อร้องเรียนในกล่อง
  • กดที่วันเพื่อดูรายละเอียดด้านล่าง
  • กดที่กล่องข้อร้องเรียน → ลิงก์ไปหน้ารายละเอียดเรื่องนั้น

5.2 การนัดประชุม

1กดปุ่ม "+ นัดประชุม" มุมบนขวา

2กรอกข้อมูลในกล่องลอย:

  • หัวข้อเรื่อง
  • สถานที่
  • วันที่ (ปฏิทิน พ.ศ.)
  • เวลา

3กดปุ่ม "บันทึก"

แจ้งเตือน LINE: เมื่อนัดประชุมใหม่ ระบบจะส่งแจ้งเตือนไปยัง LINE OA โดยอัตโนมัติ

5.3 การแก้ไข/ลบนัดประชุม

1กดที่วันที่ในปฏิทินที่มีนัดประชุม

2ในรายการด้านล่าง จะแสดงปุ่ม "แก้ไข" และ "ลบ"

3กดแก้ไข → แก้ข้อมูลในกล่องลอย → กดบันทึก

6. การใช้งาน Dashboard

Dashboard แสดงภาพรวมสถิติข้อร้องเรียนทั้งหมดในระบบ

6.1 กล่องสถิติหลัก

กล่องข้อมูลกดเข้าดู
ข้อร้องเรียนทั้งหมดจำนวนทั้งหมดในระบบลิงก์ไปทะเบียน (ทั้งหมด)
แก้ไขแล้วจำนวนที่ดำเนินการเสร็จ + เปอร์เซ็นต์ลิงก์ไปทะเบียน (สถานะแก้ไขแล้ว)
รอดำเนินการจำนวนที่ยังไม่เสร็จลิงก์ไปทะเบียน (สถานะรอดำเนินการ)
ระยะเวลาเฉลี่ยจำนวนวันเฉลี่ยในการจัดการ-

6.2 กราฟและแผนภูมิ

  • กราฟแท่งรายเดือน — จำนวนข้อร้องเรียนทั้งหมด, แก้ไขแล้ว, รอดำเนินการ
  • แผนภูมิวงกลม — สัดส่วนตามหน่วยงาน (แสดงชื่อหน่วยงาน)
  • กราฟแท่งแนวนอน — จำนวนตามระดับความรุนแรง
  • กราฟเส้น — แนวโน้มรายเดือน
เคล็ดลับ: กดที่กล่องสถิติด้านบนเพื่อลิงก์ไปหน้าทะเบียนที่กรองสถานะให้อัตโนมัติ

ภาคผนวก: ข้อมูลทางเทคนิค
